# 自定义视频录制页面设计

## 背景

当前小程序视频录制使用 `wx.chooseMedia` 调用系统原生相机，存在以下问题：

- 不同品牌设备 UI 不一致，用户体验差
- 部分品牌设备无法调起录像功能，直接报错"录制失败"
- 无法在录制过程中添加引导提示
- 缩放操作不便，用户不知道如何拉近拍清细节
- 录制参数（分辨率、时长等）不可控

## 技术方案

采用 **`<camera>` 组件 + `CameraContext`** 方案，使用微信小程序内置 camera 组件替代 `wx.chooseMedia`，实现完全自定义的录制界面。

选择理由：
- `<camera>` 组件从基础库 1.6.0 起支持，覆盖率 ~99%
- 支持 `pauseRecord/resumeRecord`（基础库 2.7.0+）
- 支持 `setZoom` 编程式缩放（基础库 2.10.0+）
- 不依赖系统相机 App，避免"无法调起"问题

## 页面结构

从底到顶 4 层叠加：

```
┌─────────────────────────────┐
│         导航栏（自定义）        │  返回按钮 + 标题"录制事故视频"
├─────────────────────────────┤
│                             │
│      <camera> 全屏预览        │  占满剩余空间，resolution="high"
│                             │
├─────────────────────────────┤
│  引导提示文案区域              │  半透明背景，录制5秒后淡出
│  ✅ 角度找好，左右拍全，拉近距离  │
│  ✅ 记得拍清品牌和车牌          │
├─────────────────────────────┤
│  缩放滑条 1x ────●──── 5x    │
│       ● 00:00:05  录制中      │  计时器 + 状态指示
│  [切换摄像头]  (● 录制按钮)     │  底部控制栏
└─────────────────────────────┘
```

关键属性：
- `<camera>` 组件：`device-position="back"`，`resolution="high"`（720p），`flash="off"`
- 页面 `navigationStyle: "custom"`，自定义导航栏

## 状态流转

5 个状态：`idle` → `recording` ⇄ `paused` → `preview` → `uploading`

```
idle ──(点击录制)──▶ recording ──(点击暂停)──▶ paused
                     ▲    │                    │
                     │    │(点击继续)            │
                     │    └────────────────────┘
                     │    │
                     │    │(点击停止，时长≥10s)
                     │    ▼
                     │  preview ──(点击上传)──▶ uploading
                     │    │
                     └────┘(点击重录)
```

各状态 UI 行为：

| 状态 | 底部控制栏 | 导航栏 | 引导提示 | 缩放滑条 |
|------|-----------|--------|---------|---------|
| idle | 录制按钮 + 切换摄像头 | 显示 | 显示 | 显示 |
| recording | 暂停 + 停止 + 计时器 | 隐藏 | 前5秒后淡出 | 显示 |
| paused | 继续 + 停止 + 计时器闪烁 | 隐藏 | 隐藏 | 显示 |
| preview | 重新录制 + 上传 | 显示 | 隐藏 | 隐藏 |
| uploading | 禁用，显示进度 | 禁用返回 | 隐藏 | 隐藏 |

关键交互：
- 停止录制时若 <10秒，toast 提示并继续录制
- 达到5分钟自动停止，进入 preview
- 录制中点返回需二次确认
- 暂停状态计时器数字闪烁

## 缩放控制

- `<slider>` 组件，min=10，max=50（代表 1.0x~5.0x）
- `bindchanging` 实时调用 `CameraContext.setZoom(value / 10)`
- 滑条上方居中显示当前倍数 "2.3x"
- 切换摄像头时重置为 1x
- 样式：半透明圆角条，白色滑块，黄色已选区域

## 摄像头切换

- 底部控制栏右下角翻转图标
- 切换 `device-position`（back ↔ front）
- 切换时重置缩放为 1x
- idle 和 recording 状态可用

## 预览与上传

preview 状态复用现有逻辑：
1. `<video>` 播放 tempFilePath，controls + autoplay
2. 重新录制：回到 idle 状态
3. 上传：获取位置 → uploadFile → 调用 `/api/v1/wx/accid/newAdd` → 订阅消息 → navigateBack
4. 上传中：全屏遮罩 + 环形进度条

## 兼容性与降级

三级降级策略：

| 级别 | 条件 | 体验 |
|------|------|------|
| 完整 | camera 正常 + pauseRecord 支持 | 全部功能 |
| 部分 | camera 正常 + pauseRecord 不支持 | 隐藏暂停按钮，仅开始/停止 |
| 降级 | camera 初始化失败 | 提示 + 回退到 wx.chooseMedia |

检测方式：
- `binderror` 捕获 camera 初始化失败
- `wx.canIUse('CameraContext.pauseRecord')` 检测暂停支持
- `wx.canIUse('CameraContext.setZoom')` 检测缩放支持

权限处理：
- `wx.authorize({ scope: 'scope.camera' })` 提前请求
- 被拒绝时 `wx.openSetting()` 引导开启

## 录制参数

| 参数 | 值 |
|------|------|
| 分辨率 | 720p（resolution="high"） |
| 最短时长 | 10秒 |
| 最长时长 | 5分钟（300秒） |
| 默认摄像头 | 后置 |
| 缩放范围 | 1x ~ 5x |
